# 技术博客排版规范（Agent 指南）

## 1. 执行边界

- 默认只整理格式并纠正明确的错别字。
- 不得擅自改写观点、事实、论证、示例、引用或代码语义。
- 用户明确要求修改内容时，以用户指令为准。
- 不确定是否属于错误时，保留原文并向用户说明。

## 2. Frontmatter

Frontmatter 必须从文件第一行开始。标准模板：

```yaml
---
title: '文章标题'
date: 2026-04-23
tags:
  - 标签1
  - 标签2
categories: ['note', 'ai']
knowledge: ['ai/nn-math']
maturity: 基础
lang: zh
---
```

执行规则：

- `title`、`date` 必填。
- `categories`、`knowledge` 必须使用单行内联数组。
- `tags` 必须使用两空格缩进的 YAML 多行列表。
- 字符串使用单引号；日期使用裸值 `YYYY-MM-DD`。
- `categories` 只能使用 `src/data/categories.json` 中存在的完整 slug 路径。
- `knowledge` 只能使用 `src/data/knowledge.json` 中存在的完整 slug 路径。
- 禁止猜测或新造 slug。
- `maturity` 只能是 `基础`、`当下热点`、`未来展望`。
- `lang` 只能是 `zh` 或 `en`。
- 原文已有 `description`、`image`、`updated`、`draft` 等可选字段时不得擅自删除。
- 新文章优先使用 `npm run new-post <文件>` 生成 frontmatter。

日期按以下优先级确定：用户指定日期 > 原文明示日期 > 可可靠推断的日期 > 文件最后修改日期 > 当前日期。

## 3. 正文结构

- 页面 H1 由 frontmatter 的 `title` 生成；正文使用 `##` 至 `######`（H2–H6），不得再写 `#`。
- 标题逐级递进，不得跨级；标题不得添加序号。
- 分隔线统一使用 `---`，前后各留一个空行。
- 列表项之间不留无意义空行；嵌套列表保持一致缩进。
- 列表标签与内容写在同一行。
- 仅在能确认是粘贴或转换残留时，删除孤立标点、重复文本和残缺 Markdown 标记。
- 只纠正能够确认的错别字和技术术语拼写错误。

## 4. 公式

- 行内公式使用 `$...$`；块级公式使用 `$$...$$`。
- 多字符上下标必须写为 `_{...}`、`^{...}`。
- 数学模式中的希腊字母使用 LaTeX 命令，如 `\mu`、`\sigma`、`\pi`。
- 多行公式使用 `aligned` 或 `\\` 换行。
- 函数名使用标准命令或 `\mathrm{...}`，如 `\max`、`\mathrm{clip}`。

## 5. 图片

- 图片文件统一放在 `public/images/posts/`。
- `src/content/posts/zh/` 内的 Markdown 使用相对路径：`../../../../public/images/posts/xxx.png`。
- HTML `<img>` 使用同一相对路径。
- 不得只写 `/images/posts/xxx.png`，否则 Typora 无法按项目文件定位图片。

## 6. 文本流程图

- 新增文本流程图统一使用 `text` 围栏；现有 `txt`、`plaintext` 围栏可保留。
- 流程图必须包含可识别结构：框线或分支字符，或至少两个箭头。
- 箭头长度由结构决定；为对齐同层节点、维持列关系或表达跨列连接时可以使用长箭头。
- 禁止用长箭头做无结构意义的填空或拉宽；不影响对齐与可读性时使用更短的连接。
- 不得为适配手机端手动拆断节点、箭头或连接线。
- 移动端只允许流程图代码块内部横向滚动，不得造成页面级横向溢出。
- 自动识别逻辑以 `src/plugins/shiki-text-diagram.mjs` 为准，显示规则以 `src/styles/global.css` 为准。

## 7. 桌面博客响应式布局基准

- 参考视口宽度为 `1749px`；正文范围为 `215–1336px`，目录范围为 `1366–1645px`，两者之间保留 `30px` 间距。
- 四条边界按完整视口宽度等比缩放，对应 `12.292738708vw`、`76.386506575vw`、`78.101772441vw`、`94.053744997vw`；必须使用视口单位排除不同滚动条宽度的影响，不得退回固定像素位移或包含块百分比。
- 正文和目录使用 `1121fr / 279fr` 网格，列间距为 `1.715265866vw`（参考视口下 `30px`）；桌面规则从 `1024px` 开始生效，低于该宽度继续使用移动端目录。
- 桌面正文、标题、H2–H6、元信息和目录字体使用 `clamp(px, px + vw, px)` 缩放；在 `1749px` 参考宽度下，正文约为 `16px`、标题约为 `36px`、H2 约为 `30px`。
- 未获用户明确授权，不得在其他问题调试中修改 `.post-layout-article`、`.post-header-row`、`.post-article-header`、`.post-article-body` 或 `#toc-sidebar` 的范围和位置。
- 回归检查位于 `tests/mobile-overflow-regression.mjs`，覆盖 `1024、1280、1440、1749、1920px`；边界比例或字体缩放漂移都必须使回归失败。
- 手机目录球进度以正文阅读区间为基准：正文顶部到达页头下方的阅读线时为 0%，正文底部到达同一阅读线时为 100%；若页面无法继续滚动到理想终点，则以视觉视口能到达的页面最底部为 100%。正文底部仅进入视口底边时必须仍小于 100%。
- 阅读坐标必须使用 `visualViewport.pageTop/height` 兼容地址栏、软键盘与缩放变化；SVG 弧长必须按 `getTotalLength()` 的实际圆周计算，不得用内部进度值代替真实弧长验证。真实进度小于 100% 时，36px SVG 进度环至少保留约 2px 可见缺口，只有到达完成终点才允许闭环。
- 32px 手机目录按钮不得绘制会与进度弧混淆的完整圆形边框或阴影；36px SVG 在四边各超出按钮 2px，使用 1px 描边并保持贴球样式。
- 目录球进度回归位于 `tests/reading-progress.test.mjs` 和 `tests/inspect-reading-progress.mjs`，必须覆盖旧完成终点不得满环、未完成时的最小可见缺口、`Reward model 与偏好概率` 定点开环、布局视口与视觉视口高度不等及视觉视口动态变化。

## 8. 发布检查

1. 检查 frontmatter 位置、格式、枚举值及 slug 是否有效。
2. 检查正文标题位于 `##` 至 `######`，从 `##` 开始且没有跨级。
3. 检查公式、列表、分隔线、图片路径和残缺 Markdown。
4. 运行 `npm run check:mobile-overflow`；必须通过全站移动端检查、极端内容夹具和桌面博客布局锁定。
5. 运行 `npm run check:markdown`。
6. 运行 `npm run build`。
