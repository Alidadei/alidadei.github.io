# 知识架构分类

> 每篇文章用**三个正交维度**标注,互不重叠、各不腐烂。本文是设计说明 + 维护指南;代码侧由 `src/data/knowledge.json`(主题树)+ 文章 frontmatter 的 `category`/`knowledge`/`maturity` 字段驱动。

---

## 三个正交维度

| 维度 | 含义 | 取值 | 腐烂? |
|---|---|---|---|
| **origin**(产出方式) | 我怎么得到这个知识 | 学习笔记 / 个人实践 | 不会 |
| **subject**(知识主题) | 关于什么 | 主题树节点(见下) | 不会(transformer 永远是 transformer) |
| **maturity**(时效定位) | 在时间轴上的位置 | 基础 / 当下热点 / 未来展望 | 不会,锚在文章 `date` 上 |

**为什么是三个、且正交:**
- origin 与 subject 容易混(学习 vs 主题)——它们垂直:origin 是"怎么来的",subject 是"关于什么"。
- subject 与 maturity 容易混(LLM 既是主题又是"热点")——也垂直:subject 是"什么主题"(稳定),maturity 是"现在算基础还是热点"(随时间变)。
- 把 maturity 当 subject 的子节点(基础/大模型/未来)会让主题树**腐烂**:2025 的"热点"≠ 2026 的"热点"。抽成独立维度、锚在 date 上,就不腐烂。

**时效为什么不腐烂**:`maturity` 绑在文章 `date` 上,而日期永远显示着。一篇 2025-06 标 `当下热点` 的文章,读者一看日期就懂是"2025 年中的热点"——自带时间上下文,无需额外字段。

**沉淀(graduation)**:概念经时间检验成为奠基性 → 把相关文章的 maturity 从 `当下热点` 改成 `基础`(subject 不动)。这是 maturity 维度的重标注,不碰主题树。

## 维护方式

### 数据源
- **`src/data/knowledge.json`** — 主题树骨架(domain>topic,slug + 中英标签)。**唯一需要偶尔维护的结构文件。**
- 文章 frontmatter(三轴):
  ```yaml
  categories: ['note','ai','transformer']  # 既有:第 0 段 note/practice = origin(学习/实践),供时间线 tab 用
  knowledge: ['ai/llm/rl']                 # 新增:subject 主题路径(存 slug,可多个,交叉归档)
  maturity: 当下热点                        # 新增:时效 基础 / 当下热点 / 未来展望
  date: 2026-06-15                         # maturity 的时间锚(已存在)
  ```
  > **阶段性**:origin 暂复用 `categories[0]`,时间线 tab 仍读 `categories`;待时间线 tab 改读 `knowledge` 后可废弃 `categories`,彻底单源。

### 写新文章(一次引导式操作)
`npm run new-post`(交互式脚手架,沿用 `cms.mjs` 风格)依次问:
标题 → origin(↑↓ 学习/实践)→ subject(空格多选主题树节点,显示中文名)→ maturity(↑↓ 基础/当下热点/未来)→ 文件名
→ 生成带 frontmatter 的 `.md`,然后只写正文。三个标签一次填完,不用记 slug、不会拼错、不会漏。

### 安全网:构建时强校验
`npm run build` 检查每篇:
- `category` ∈ {note, practice}
- `knowledge` 每项存在于 `knowledge.json`
- `maturity` ∈ {基础, 当下热点, 未来展望}

漏填/拼错 → **构建失败并指名哪篇哪个字段**。绕过脚手架手写也 publish 不了错标签文章。

### 沉淀流程
定期扫 `maturity = 当下热点` 的文章 → 问"是否已奠基、几年后仍成立" → 是则改 maturity 为 `基础`(subject 不变)。

### 重构时机
- 某 subject 节点 > ~8 篇且内部明显聚类 → 拆节点(如 Agent → Agent 理论 / Claude Code)。
- 全新方向 → `knowledge.json` 加节点。
- 未来展望里的方向成熟 → maturity 迁到 基础 / 当下热点。

### 视图呈现
- **知识树视图**:按 subject 主题树展开;用 maturity 当**透镜**(按时效分区块,或节点上不同色/徽章)。基础/热点/未来是"罩在主题树上的滤镜",不是固定节点 → 不会把不同年份的热点混进同一个坑。
- **时间线视图**:按时间;maturity/origin 可作为筛选或徽章。
