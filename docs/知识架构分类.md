# 知识架构分类

> 每篇文章用**三个正交维度**标注,互不重叠、各不腐烂。代码侧:`src/data/knowledge.json`(主题树)+ 文章 frontmatter 的 `categories[0]`(origin)/ `knowledge`(subject)/ `maturity`(时效)。

## 三轴

| 维度 | 含义 | 取值 | 来源 |
|---|---|---|---|
| **origin** 产出方式 | 怎么得到的 | 学习笔记 `note` / 个人实践 `practice` | frontmatter `categories[0]`(复用既有字段) |
| **subject** 知识主题 | 关于什么 | 主题树节点路径 | frontmatter `knowledge: ['ai/llm/rl']` |
| **maturity** 时效定位 | 时间轴上的位置 | `基础` / `当下热点` / `未来展望` | frontmatter `maturity`,锚在 `date` |

三轴正交、各管一摊。**maturity 锚在文章 date 上**(读者看日期就懂「2025 当下热点」指什么),所以不腐烂——这是它独立成轴、不当主题树子节点的原因。**沉淀**= 把文章 maturity 从 `当下热点` 改成 `基础`(subject 不动)。

## 主题树(subject)

```
AI ├─ 大模型 ├─ 强化学习   (RLHF/GRPO/Agentic RL)
│          └─ Agent       (agent 系统、agentic 工具)
   └─ 神经网络数学          (softmax/归一化/temperature)
编程 ├─ 数据结构 / Git
电子信息 └─ 嵌入式
```

- 存英文 slug(`ai/llm/rl`),显示查表取中英 label → **改名只改 `knowledge.json` 的 label,文章不动**。
- 节点 > ~8 篇且内部聚类 → 拆;全新方向 → 加节点;未来展望里的方向成熟 → maturity 迁到 基础/当下热点。

## 维护

**写新文章**:`npm run new-post` 交互式问 标题 / origin / subject / maturity / 文件名,自动生成 frontmatter。

**frontmatter 三行**:
```yaml
categories: ['note']          # origin(同时是时间线分类)
knowledge: ['ai/llm/rl']      # subject 主题路径(可多个,交叉归档)
maturity: 当下热点            # 基础 / 当下热点 / 未来展望
```

**安全网**:`npm run build` 强校验——knowledge 路径必须存在于 `knowledge.json`、maturity 必须合法,否则构建失败指名哪篇。手写也 publish 不了错标签文章。

**沉淀**:定期扫 `当下热点` → 已奠基、几年后仍成立则改 `基础`(subject 不变)。

## 视图

- **知识树**:按 subject 主题树展开,maturity 当透镜(基础/热点不同色点);节点可折叠。「时间线 / 知识树」可切换,localStorage 记住选择。
- **时间线**:按时间;origin(`categories[0]`)驱动分类 tab。
