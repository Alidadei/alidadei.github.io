### 移动端表格

根因不是横向滚动失效。表格本身已有 `overflow-x: auto`，真正冲突是：移动端规则只给 `th/td` 设置了 `white-space: nowrap`，而文件后方的全局行内代码规则用 `!important` 给 `<code>` 重新设置了 `white-space: normal`、`overflow-wrap: anywhere` 和 `word-break: break-word`。第二列恰好主要是行内代码，所以父单元格不换行、内部代码却会换行。

修复已限制在 `max-width: 640px`：表格单元格和全部后代统一禁止自动断行，超出宽度继续由表格自身横向滚动承载。桌面文章标题、正文和目录的锁定范围没有修改。

此前回归只验证“页面没有横向溢出或裁切”，没有验证“表格后代必须保持整行”，所以一个视觉错误可以在页面宽度仍正确时漏过。现在测试同时检查真实文章的表格后代，并加入含 `log(0.42) ≈ -0.87` 行内代码的极端夹具。Microsoft Edge 全站实测覆盖 40 个路由、320/360/390/430px 共 160 次页面检查和 4 次极端夹具检查，失败均为 0；1280px 桌面布局锁也通过。

### 防复制与安全

公开静态站点无法做到“代码被 clone 后技术上绝对不能用”：所有前端判断都随代码交给复制者，可以删除。真正可执行的边界是私有源码、服务端鉴权和不公开密钥。

当前 CMS 已有 GitHub 用户 ID 白名单；本轮进一步修复了 Worker 反射任意 `Origin` 的问题，只允许官网和本机开发来源，陌生克隆站的浏览器请求会在路由入口收到 403。无 `Origin` 的 CI/Worker 请求继续可用，敏感 API 仍由会话鉴权负责；`.gitignore` 也已补齐 `.env*` 与 Cloudflare `.dev.vars*`。4 项 CORS 单元/路由测试与 Worker TypeScript 打包检查均通过。该 Worker 改动还没有部署，不会冒充已经在线生效。

完整边界、当前审计、无效方案和后续优先级见 `docs/网站防复制与安全边界.md`。如果最终目标是让别人拿不到可直接运行的源码，最有效的下一步仍是把源仓库设为私有，只发布构建产物；公网成品页面依然无法禁止抓取。
