# 云、场景物体与太阳景深同步验证

- 日期：2026-07-20
- 分支：`feat/cloud-sun-depth-sync`
- 浏览器：Microsoft Edge 149.0.4022.80
- 桌面遮挡截图：`cloud-sun-occlusion-edge.png`
- 手机端截图：`cloud-sun-mobile-edge.png`

## 实现约束

- 太阳圆盘、光晕、渐变和阴影仍由原 DOM/CSS 绘制，样式参数未改变。
- 太阳整体上抬量恢复为历史值 `50px`。
- 遮挡源不再限定为云朵轮廓：Three.js 每帧把太阳周围 `128 × 128` 区域内的完整场景渲染到带深度缓冲的离屏目标，再把遮挡像素转为 SVG 蒙版。因此树、机器人、漂浮物、云和地面等任何位于太阳前方的场景网格都会遮挡太阳。
- 太阳的屏幕位移跟随两个固定云层锚点的平均屏幕位移；云朵自身的缓慢漂移动画不会拖动太阳。
- 远端太阳角度按 `场景半径 / 太阳半径` 缩小：桌面为 `350 / 1200 ≈ 29.2%`，手机为 `480 / 1200 = 40%`。由 `r_cloud × Δθ_cloud = r_sun × Δθ_sun`，远端角度更小而移动弧长与云层一致。

## Edge 自动回归结果

- 默认相机下，太阳中心与原 CSS 轨迹中心误差：桌面 `(0px, 0px)`，手机 `(0px, 0px)`。
- 固定时间 07:18、默认俯视角 `0.35` 时，场景轮廓覆盖太阳圆盘约 `50.34%`，截图可见云边缘遮挡太阳。
- 继续向下俯视后，太阳 Y 位移为 `-157.171875px`，云层基准 Y 位移为 `-157.17811461279834px`；位移比例为 `0.9999603`。
- 页面滚动 500px 并触发时间更新后，太阳没有跟随正文滚动跳变。
- 太阳圆盘仍为 `16 × 16px`，原渐变、阴影和光晕断言全部通过。
