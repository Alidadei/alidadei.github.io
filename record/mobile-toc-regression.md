## 移动端故障复现

- 页面：`/zh/blog/llm-post-training-basics-and-jargon/`
- 设备视口：390px
- 修复前：视觉视口为 390px，但长 KaTeX/MathML 公式把布局视口撑到 701px；目录按钮右边缘位于 689px，而预期位置为 378px。
- 对照页：`/zh/blog/claude-code-dynamic-workflows/` 不含该类长公式，布局视口保持为设备宽度。

## 桌面端故障复现

- 页面：`/zh/blog/llm-post-training-basics-and-jargon/`
- 视口：1280 × 900px
- 修复前连续发送 5 次 `500px` 向下滚轮事件，页面位置为 `0 → 342 → 321 → 525 → 327 → 321`，无法持续向下阅读。
- 原因：根元素上的 `overflow-x: hidden` 形成额外滚动容器；目录高亮使用 `scrollIntoView()` 时会连带调整文档滚动位置。

## 移动端二次复测

- 第一版回归只校验了根元素宽度与按钮坐标；根级 `overflow-x: clip` 将 `documentElement.scrollWidth` 压到 390px，但 `body.scrollWidth` 仍为 701px，因此不能代表所有真实手机浏览器都不会采用被撑宽的页面坐标系。
- 进一步定位到 KaTeX 的屏幕阅读器 MathML 层仍会把文章内容行撑宽。移动端现由 `.post-content-row` 在内容层限制横向溢出，不再依赖根元素掩盖问题；公式、表格和代码块仍在各自容器内横向滚动。
- 移动端测试新增强制条件：`body.scrollWidth` 必须等于设备视口宽度。

## 修复后验证

- `html` 与 `body` 使用 `overflow-x: clip`，文章内容行在 1024px 以下额外使用兼容性更广的 `overflow-x: hidden`；目录高亮仅通过目录导航自身的 `scrollTo()` 保持可见。
- 两篇目标文章在 320、360、375、390、412、430、667、844、932、1023px 下共验证 20 组，全部通过。
- 每组移动端测试均校验：根布局视口等于设备视口、目录按钮距屏幕右侧 12px、打开后的目录面板右边缘等于屏幕右边缘。
- 当前 16 篇非重定向中文文章在 390px 下全部通过，且每篇的 `body.scrollWidth` 均严格等于 390px；无 `h2`–`h4` 标题的文章会隐藏无效目录控件。
- 两篇目标文章在 1024、1280、1440、1920px 下共验证 8 组；每组 5 次滚轮事件后的滚动位置均为 `0 → 500 → 1000 → 1500 → 2000 → 2500`。
- `npm.cmd run astro -- build` 成功，生成 45 个页面；外部友链 feed 在受限网络中出现非致命 fetch warning。

验证脚本：`tests/inspect-mobile-toc.mjs`、`tests/inspect-desktop-scroll.mjs`
