# 桌面博客响应式布局验证记录

- 日期：2026-07-22
- 验证页面：`/zh/blog/llm-post-training-basics-and-jargon/`
- 参考视口宽度：`1749px`
- 目标坐标：正文左边界 `245px`；正文右边界/目录左边界 `1366px`；目录右边界 `1645px`

## 实现基准

- 正文与目录整体范围：`14.008004574vw` 至 `94.053744997vw`。
- 正文与目录列宽比：`1121fr / 279fr`。
- 桌面生效断点：`1024px`。
- 正文、标题、H2–H6、元信息和目录字体使用 `clamp(px, px + vw, px)`。

## 百分比方案失败记录

初版使用包含块百分比。Edge 在 `1749px` 视口下实测为：

| 边界 | 目标 | 百分比方案实测 |
| --- | ---: | ---: |
| 正文左边界 | 245px | 244.15625px |
| 正文右边界 / 目录左边界 | 1366px | 1361.296875px |
| 目录右边界 | 1645px | 1639.328125px |

根因是页面的 `6px` 垂直滚动条使包含块宽度小于 `window.innerWidth`；百分比按扣除滚动条后的宽度计算。改用 `vw` 后，边界直接按完整视口宽度计算。

布局误差阈值为 `0.1px`，字体误差阈值为 `0.12px`。

| 浏览器 | 版本 | 桌面宽度 | 结果 |
| --- | --- | --- | --- |
| Microsoft Edge | 150.0.4078.83 | 1024、1280、1440、1749、1920px | 5/5 通过 |
| Google Chrome | 150.0.7871.130 | 1024、1280、1440、1749、1920px | 5/5 通过 |

在 `1749px` 视口下，字体计算值为：正文 `15.9996px`、页面标题 `35.9987px`、H2 `30.0009px`、一级目录项 `13.4979px`。

移动端同时复查了 `320、360、390、430px`：页面溢出、正文裁切和极端内容夹具均为 `0` 个失败。
