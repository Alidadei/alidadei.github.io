## 中转站机制速览(选型前必懂)

> 来源:helpaio《Claude Code 中转站选购指北》。

### 工作原理

```
你的请求 → 中转站 → Anthropic 官方 → 中转站 → 返回结果
```

中转站本质是 HTTP 中间人代理。**理论上中转方可见你全部请求内容**(代码、prompt)。涉密代码不建议走中转。

### 渠道四分类(同一站点常并存多个分组)

| 渠道 | 来源 | 参考价(元/刀) | 关键特征 |
|---|---|---|---|
| **官方** | Max 20x 号池 / 官方 API | 0.85–1.5 | 模型完整、原生 Prompt Caching、1M 上下文 |
| **一方合作** | AWS / Azure / GCP | 中 | 性能稳、社区反馈降智概率低 |
| **逆向** | Kiro / Cursor / Windsurf 等客户端提取 | 0.1–0.3 | 便宜但降智、上下文缩水(200K/128K)、工具调用易失败、可能假缓存 |
| **试用/临时** | 官方试用额度 | 极低 | 不稳定 |

> **官方与逆向价差可达 5–6 倍。** 买前必须问清是哪类渠道,低价基本等于逆向。

### 倍率与真实价格

中转站默认 `1 元 = 1 美元`(非真实汇率)。倍率是加价系数:

| 倍率 | Sonnet 4.6 输入(元/M tok) | 含义 |
|---|---|---|
| 1× | 3 | 名义价 |
| 1.5× | 4.5 | 2026 年多数一线站现状 |
| 7× | 21 | 接近真实汇率 |

> 2026-04 起 Anthropic 加强 Claude KYC 风控,多家站 Claude 分组从 1.2×→1.5× 甚至 2.0×。**当前 Claude 低价位(≤1×)几乎都伴随风险信号。**

### 价格四组成 + 缓存率(成本关键)

实际费用 = 输入 + 输出 + 缓存创建 + 缓存读取。其中:

- 输出 = 输入的 **5 倍**
- 缓存读取 = 输入的 **0.1 倍**(命中缓存后输入成本降到 1/10)
- 缓存创建(5min)= 输入的 1.25 倍

**缓存命中率是隐形大头**:一线站 Claude Code 缓存率 80–95%;若某站缓存率仅 60%,频繁重建缓存会让"低价"实际更贵。**缓存率也是判真伪的辅助信号** —— 逆向/假渠道常不支持原生缓存,或站长写死一个假的高缓存率。

## 行业风险全景

### 掺假四种方式

| 类型 | 说明 | 本工具能否检出 |
|---|---|---|
| 官方降级 | 官方不可用时临时切逆向 | ✅ 审计探针可发现 ratio 异常 |
| 混合渠道 | 官方与逆向按不确定比例混用 | ✅ 多轮审计可发现 |
| **模型替换** | 换成 GLM / DeepSeek / Gemini 冒充 | ✅ `model_echo` + 指纹探针 |
| 同系降级 | 用低版本冒充高版本(如 opus-4-5 冒充 4-6) | ⚠️ 部分能检出(指纹差异) |

> 实锤案例:DuckCode 2026-01 被发现在 1.3× Max 分组用逆向(LINUX DO 实锤帖);HongMaCC 2026-03 起全面转逆向,调用日志出现 Kimi;Privnode CC 分组掺 Kiro 逆向。

### 站长处置态度(出事后)

| 等级 | 表现 |
|---|---|
| 较好 | 主动发现、公告、退款 |
| 一般 | 问起才坦承或沉默 |
| 较差 | 归咎上游 |
| 恶劣 | 后台替换,发现者被踢出群 |

### 资金 / 跑路风险

- **跑路高频**:2026 年初 Privnode(退款要求实名却仍不退)、88Code(持续宕机数月,软跑路,推 reclaude 让老用户 8 折转余额)。
- **背景风险**:虎嗅《Token 经济》指出 —— 极低倍率(如 0.6×)背后可能是**盗刷信用卡跑出来的账号**,或**逆向 claude.ai 网页版模拟 API**。每一个低倍率分组都对应一个灰色上游。
- **建议**:少量多次充值,单次不超一周用量;优先公司运营 / 长运营期 / 有客服群的站。

### 隐私

中转 = 中间人。**所有对话对中转站透明**(V2EX 实测帖三大坑之一)。不要传涉密代码、个人敏感信息、生产 key。

## 检测工具盘点(竞品 / 参考)

本人正在使用一个工具进行模型校验，但是当前项目不是唯一做这事的开源/工具，因此agent顺带着调研了相关的工具，以便综合对比能力边界:

| 工具 | 入口 | 侧重点 |
|---|---|---|
| **当前使用： claude-detector** | github.com/7836246/claude-detector | 19 探针黑盒检测 + Token 审计双线 + 可自托管批量脚本 |
| **Veridrop** | veridrop.org/claude | thinking signature 加密验证(中转站无法伪造)+ Token 虚报 + 10 项辅助 |
| **cctest.ai** | cctest.ai | 黑盒检测掺假/降智/套壳,数十家中转站用 |
| **Ofox AI 验真** | ofox.ai/verify | 7 项自动化检测,15 秒出结果 |
| **IFFS** | iffs.cn | Claude 模型检测,速度快 |
| **aio-coding-hub** | (helpaio 出品) | 内置模型验证功能 |
| **Skill 测评法** | 虎嗅 | 726 道公开 benchmark 题对比(重,非实时) |

> **thinking signature 验证**(Veridrop 思路)值得本项目借鉴 —— Anthropic 服务端对 thinking 做了加密签名,中转站无法伪造,是比 model_echo 更强的真伪证据。

## 选型建议

**个人开发者(性价比优先):**
- 主:**PackyCode** 或 **MiCu**(一线、退款信誉好、CC 缓存率高)
- 备:不同渠道类型(如主用 Max20x,备选 AWS/Azure),避免同源故障
- 兜底:**RightCode** 或 **NekoCode**

**企业 / 团队(稳定优先):**
- **PackyCode**(可定制团队)/ **YesCode** / **Yunwu**(非 CC 场景)

**避雷底线:**
- 不碰 Privnode / 88Code / HongMaCC
- 不囤大额(单次 ≤ 1 周用量)
- Claude 倍率 < 1× 的分组,默认怀疑逆向/灰产上游,用本工具实测再决定
